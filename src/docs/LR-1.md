# Лабораторная работа №1

**Тема:** Создание и развертывание статического сайта на базе MkDocs с публикацией на GitHub Pages

## Цель работы
Освоить процесс создания статического сайта с использованием генератора документации MkDocs, научиться организовывать структуру проекта и развертывать его на GitHub Pages.

## Задание
1. Создать публичный репозиторий на GitHub.
2. Настроить GitHub Pages для публикации из каталога `/docs` ветки `main`.
3. Установить и настроить MkDocs локально.
4. Создать структуру сайта (Главная, Об авторе, Лабораторные работы).
5. Выполнить сборку и публикацию.

## Ход работы

### 1. Настройка окружения
Было создано виртуальное окружение и установлены необходимые пакеты:
```bash
python -m venv venv
source venv/bin/activate
pip install mkdocs mkdocs-material
```

### 2. Инициализация проекта
Команда `mkdocs new src` создала базовую структуру. Файл `mkdocs.yml` был настроен для использования темы `material`.

### 3. Конфигурация (mkdocs.yml)
```yaml
site_name: IR6's Portfolio
theme:
  name: material
nav:
  - Home: index.md
  - LR-1: LR-1.md
  - LR-2: LR-2.md
  # ...
```

### 4. Сборка и деплой
Сборка сайта в папку `docs` для GitHub Pages:
```bash
mkdocs build -d ../docs
```

## Выводы
В ходе работы был успешно развернут статический сайт. Использование MkDocs позволяет быстро создавать профессионально выглядящую документацию из простых Markdown-файлов. Механизм GitHub Pages обеспечивает бесплатный и надежный хостинг для портфолио.
