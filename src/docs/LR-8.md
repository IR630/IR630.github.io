# Лабораторная работа №8

> **Тема:** Веб-скрапинг. Извлечение данных с сайта news.itmo.ru

## Цель работы
Освоить методы веб-скрапинга: загрузку HTML-страниц, парсинг DOM-структуры, обход пагинации и сохранение результатов в CSV.

## Задание
1. Собрать список новостей с [news.itmo.ru](https://news.itmo.ru/ru) из раздела `main_news`.
2. Для каждой новости извлечь: идентификатор (из URL), название, дату размещения и ссылку на страницу.
3. Сохранить общий список в CSV-файл.
4. Для каждой новости дополнительно извлечь текст, теги и количество просмотров.

## Выполнение

### 1. Настройка и вспомогательные функции
Использованы библиотеки `requests` и `BeautifulSoup`. Реализованы функции:
- `get_soup(url)` — загрузка страницы и создание объекта `BeautifulSoup`
- `extract_news_id(url)` — извлечение числового ID из URL (`/news/14850/`)
- `extract_text(soup)` — очистка HTML от служебных элементов и сбор текста статьи

### 2. Сбор списка новостей
Функция `collect_news_list()` обходит страницы пагинации `https://news.itmo.ru/ru/main_news/{page}/`, извлекает элементы через CSS-селектор `li:has(h4 > a[href*="/news/"])` и дедуплицирует записи по ID.

| Параметр | Значение |
|----------|----------|
| Обработано страниц | 127 |
| Собрано новостей | 1131 |
| Уникальных ID | 1131 |

Между запросами добавлена случайная задержка (0.3–1.0 с), чтобы снизить нагрузку на сервер.

### 3. Сохранение общего списка
Данные записаны в `scraped_news_data.csv` со столбцами:
- Идентификатор новости
- Название новости
- Дата её размещения
- URL на страницу с конкретной новостью

### 4. Парсинг содержимого каждой новости
Функция `parse_news_details(url)` заходит на страницу отдельной новости и извлекает:

| Поле | Источник на странице |
|------|----------------------|
| Дата | атрибут `datetime` у тега `<time>` |
| Просмотры | `span.icon.eye` |
| Заголовок | `<h1>` |
| Теги | `<meta property="article:tag">` |
| Текст | `.post-content` / `.content.js-mediator-article` |

Результаты сохранены в `news_content/news_details.csv` для всех 1131 новостей.

### 5. Проверка качества данных
Проведена сводная проверка собранного датасета: все URL уникальны, заголовки заполнены (1126 уникальных названий из 1131 записей), даты охватывают период с 2016 по 2026 год.

## Стек технологий
`requests` | `beautifulsoup4` | `csv` | `re` | `pandas`

## Выводы
> **Результат:** Реализован полный пайплайн веб-скрапинга новостного портала ИТМО. Сначала с раздела `main_news` собран общий каталог из 1131 новости и сохранён в `scraped_news_data.csv`. Затем для каждой записи извлечены просмотры, текст и теги; данные сохранены в `news_content/news_details.csv`. Показано, что при парсинге важно учитывать особенности вёрстки сайта, использовать CSS-селекторы и не смешивать разные поля (например, дату и просмотры).
