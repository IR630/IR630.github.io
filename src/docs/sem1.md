# Лабораторные работы. Семестр 1

> **Тема:** Основы программирования на Python, алгоритмы и структуры данных, работа с базами данных и оптимизация.

## Цель курса
Изучение базовых концепций программирования на языке Python, алгоритмических подходов к решению задач, а также освоение принципов работы с базами данных, веб-запросами и профилированием кода.

## Список заданий

### 1. Сумма двух чисел (Two Sum)
* **[Исходный код](https://github.com/IR630/py_itmo/tree/main/sem1/laba_1)**
* **Описание:** Реализация классической алгоритмической задачи поиска индексов двух чисел в массиве, сумма которых равна заданному числу `target`. Также реализована функция "сглаживания" вложенных списков (`flatten`) и извлечения чисел из строковых данных. Написаны юнит-тесты с использованием модуля `unittest` для проверки различных граничных случаев (передача строк, сильно вложенных списков).

### 2. Бинарный поиск
* **[Исходный код](https://github.com/IR630/py_itmo/tree/main/sem1/laba_2)**
* **Описание:** Изучение основ асимптотической сложности алгоритмов (Big O). Написана функция классического бинарного поиска (O(log n)) по отсортированному массиву. Проведено практическое сравнение производительности с обычным линейным поиском (O(n)) на больших объемах данных.

### 3. Бинарное дерево
* **[Исходный код](https://github.com/IR630/py_itmo/tree/main/sem1/laba_3)**
* **Описание:** Создание структуры данных "Бинарное дерево поиска" (BST) с использованием парадигмы ООП. Реализованы базовые операции: вставка нового узла, поиск элемента по значению, обход дерева. Активно применялись встроенные коллекции Python, такие как словари `dict` и именованные кортежи `namedtuple`.

### 4. Факториал
* **[Исходный код](https://github.com/IR630/py_itmo/tree/main/sem1/laba_4)**
* **Описание:** Изучение концепции рекурсии. Написаны две версии функции вычисления математического факториала: классическая рекурсивная и циклическая (итеративная). Проведено тестирование производительности с помощью `timeit`, а результаты визуализированы на графиках через `matplotlib`.

### 5. Итеративное дерево
* **[Исходный код](https://github.com/IR630/py_itmo/tree/main/sem1/laba_5)**
* **Описание:** Развитие идеи из 3-й лабораторной работы. Реализован обход бинарного дерева поиска в глубину (DFS) и ширину (BFS) с использованием программного стека и очереди вместо классических системных рекурсивных вызовов. Это позволяет корректно обрабатывать деревья очень большой глубины без риска получить ошибку `RecursionError`.

### 6. Рекурсия vs Итерация
* **[Исходный код](https://github.com/IR630/py_itmo/tree/main/sem1/laba_6)**
* **Описание:** Глубокий анализ двух парадигм. На примере генерации бинарных деревьев проведено бенчмаркинг-тестирование рекурсивных и итеративных алгоритмов с замерами времени (`timeit`) и построением графиков (`matplotlib`). Сделаны аналитические выводы об эффективности каждого из подходов.

### 7. Декораторы и логи
* **[Исходный код](https://github.com/IR630/py_itmo/tree/main/sem1/laba_7)**
* **Описание:** Создан параметризованный декоратор `@logger`, который автоматически логирует вызовы функций: переданные аргументы, возвращаемый результат и возникающие исключения. Декоратор поддерживает различные интерфейсы вывода (`handle`), включая `sys.stdout`, `io.StringIO` и модуль `logging`. Написаны подробные тесты с помощью `unittest` для проверки корректности работы.

### 8. ООП и модели данных (Валюты)
* **[Исходный код](https://github.com/IR630/py_itmo/tree/main/sem1/laba_8)**
* **Описание:** Проектирование архитектуры приложения с использованием объектно-ориентированного программирования (ООП). Созданы классы-модели предметной области: `App`, `User`, `Currency`, `Author`. Также реализован функционал получения реальных курсов валют через API ЦБ РФ с использованием модуля `requests` и обработка данных в формате JSON.

### 9. MVC, SQLite и HTTP-сервер
* **[Исходный код](https://github.com/IR630/py_itmo/tree/main/sem1/laba_9)**
* **Описание:** Разработка полноценного веб-приложения по паттерну MVC (Model-View-Controller). Создана локальная БД `sqlite3` с реализацией основных SQL-запросов (CRUD). Поднят собственный веб-сервер с помощью встроенного модуля `http.server`, а для рендеринга HTML-страниц использован шаблонизатор `Jinja2`.

### 10. Оптимизация вычислений (Cython)
* **[Исходный код](https://github.com/IR630/py_itmo/tree/main/sem1/laba_10)**
* **Описание:** Изучение подходов к оптимизации вычислений. Написано пять различных реализаций алгоритма, включая использование статической типизации и компилируемого расширения `Cython` (файлы `.pyx`). Проведено профилирование времени выполнения с помощью `timeit`, доказавшее существенное превосходство скомпилированных `Cython`-модулей над интерпретируемым кодом на Python. Для сборки написан `setup.py`.

## Стек технологий
`Python 3.10+` | `Cython` | `SQLite` | `requests` | `unittest` | `Jinja2` | `matplotlib`
